# Method candidates

Each candidate is a complete immutable machine-readable prediction procedure. It may change execution, retrieval, validation, propagation, adjudication or release interfaces, but it must not change traditional knowledge or encode revealed answer letters, option positions, case-specific winners or copied correct-option wording.

A candidate must declare every mandatory stage, stable rule IDs, source-authority boundaries, failure statuses, parent method release, anti-answer-leak results, regression scope and unresolved conditions. Promotion requires repository-only shadow validation, method-packet delivery proof, causal-use validation, clean regression and approval. Historical predictions remain bound to their original method release.
