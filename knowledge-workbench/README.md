# 来源知识工程工作台

这里保存从冻结S00–S19派生的导航、知识卡和研究状态。它不是第二套原典，也不拥有命理事实权威。

固定边界：

- `sources/canonical/`仍是唯一冻结知识底座；训练中不改。
- `source-inventory.json`只记录结构、哈希、计数和定位，不贡献命理结论。
- `school-method-cards.json`是人工与AI共同梳理的可执行摘要；任何卡片都必须保留来源锚点、适用条件、限制、反例和能力上限。
- 来源中的命例只用于理解作者如何运用方法，不能自动成为规则正确性的独立证明。
- 训练案例的正确或错误只能更新`model-learning/`中的通用调用规则，不能反写冻结原典或把答案写进知识卡。

知识卡状态：

- `CURATED_UNVALIDATED`：完成来源梳理，尚无跨案例验证。
- `PARTIALLY_VALIDATED`：至少一个不同案例提供支持或反例，但未达通用门槛。
- `VALIDATED_WITH_SCOPE`：达到政策门槛，并明确适用范围与反例。
- `CHALLENGED`：出现足以缩小或推翻边界的反例。
- `RETIRED`：停止用于预测。

第二阶段按四批推进：紫微静态、紫微动态与流派、八字、双轨融合与专题闭合。每批都先做来源分层，再做方法卡、冲突矩阵和案例验证映射。
