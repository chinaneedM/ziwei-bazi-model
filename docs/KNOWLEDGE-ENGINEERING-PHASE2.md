# 第二阶段：来源知识梳理与模型吸收

## 目标

第二阶段不是压缩S00–S19，也不是让AI“背完”1.68亿字节后宣称掌握。目标是把原典转成四种可审计对象：来源事实、方法程序、冲突边界、案例验证证据。原典继续只读；任何AI总结都必须能回到来源锚点。

## 五层结构

1. **原典层**：S00–S19及哈希清单，唯一知识权威。
2. **导航层**：文件结构、活动控制根、来源层、方法词和案例记录统计，只用于定位。
3. **知识卡层**：每张卡固定写明主张范围、步骤、输入、适用条件、限制、反例、禁止捷径和来源锚点。
4. **案例理解层**：记录原典命例展示了作者怎样使用方法，但命例结局不自动证明规则有效。
5. **训练验证层**：只有107例未来首次盲测中预测前实际调用的规则，才可获得支持或反例；结果进入`model-learning/`证据账本。

## 每个知识点的最低字段

- 来源原子与准确定位；
- 流派或古籍归属，以及正文、注解、编辑层的区别；
- 它能够证明的最高层级；
- 必需条件、否定、例外、异文和冲突；
- 与其他知识是否同源，避免重复计权；
- 正确执行顺序；
- 常见误用和能力上限；
- 原典命例只作“用法示范”还是已获得独立训练案例支持；
- 支持案例数、反例数和仍缺的负例。

## 对用户经验与AI补充的处理

用户经验既不是默认真理，也不是需要丢弃的偏见。它先进入`USER_HYPOTHESIS`，写明观察范围、地域时代和可能遗漏的替代解释；AI提出的补充进入`AI_HYPOTHESIS`，同样不得高于来源与案例证据。二者使用相同验证门槛，谁都不能靠身份获得额外权重。

重点补足人类经验容易忽略的部分：基础发生率、选择偏差、确认偏误、地区年代差异、同人泄漏、答案本身可能有争议、反事实和负例、置信度校准、敏感属性伤害以及“不能区分”也是合法结论。
