# 推理执行层 V2

## 目标

本层检查模型是否完成了足以改变排序的实际推理，而不只检查标签、来源编号或文本长度。它不修改训练 R2、案例顺序、通过标准、数据分区或 S00–S19。

“结构—机制—时序—现实—对抗—反省”是当前工作框架，不是固定理论。后续训练可以依据跨案例证据拆分、合并、增加或淘汰维度。

## 冻结顺序

1. 在比较选项前建立一次全案共享的 `blind_chart_model`。
2. 对题干和所有选项做语义与原子化。
3. 紫微、八字分别形成独立排序并封卷。
4. 记录具体盘面事实、知识点、适用条件、反证、替代解释、证据家族和能力上限。
5. 对所有选项建立矩阵及全配对比较。
6. 执行移除关键证据并采用 Top2 最佳解释的真实反转测试。
7. 完成双轨裁决、分解置信度、规则消融和跨题一致性检查后，才允许冻结。

证据没有固定数量配额。`max_applied_rules_per_question`只限制 `model-learning` 规则装载量，不限制 S00–S19 证据检索，也不是停止思考的条件。同一底层事实必须归入同一证据家族；来源编号本身不是证据。

## Schema

- Chat 输入：`CHAT-PREDICTION-INPUT-V3`
- 预测工作簿：`PREDICTION-WORKBOOK-V2`
- 冻结预测：`FROZEN-PREDICTION-V2`
- Chat—Work 交接：`CHAT-WORK-PREDICTION-HANDOFF-V2`
- 训练 Issue：`TRAINING-ISSUE-PACKET-V3`
- 失败修正：`MODEL-LEARNING-CORRECTION-V3`
- 复训修复报告：`REPLAY-REMEDIATION-REPORT-V1`
- 推理完整性报告：`REASONING-COMPLETENESS-REPORT-V1`

机器校验以 `src/fortune_training/reasoning.py` 为权威；`schemas/`提供外部结构入口。

## 失败修正与规则治理

失败先分类根因，再选择修正类型。执行门、测量、校准、权重、范围、合并、退休、测试或待验证假设均可形成新的模型思路；只有 `NEW_GENERAL_RULE` 能增加规则目录。

决定性规则必须在预测前声明，并保存移除后的排序。移除后 Top1 不变的规则不能登记为决定性。退休规则不得装入预测上下文。

## 置信与维护

总置信度不得高于九项组件中的最弱项。首次盲测持续记录分箱准确率、Brier Score、组件校准和高置信错误；样本少于配置阈值时标记 `INSUFFICIENT_SAMPLE`，不产生虚假精确结论。

维护报告检查盲态模型、双轨封卷、证据条目与家族、全选项比较、反转、规则消融、高置信未闭合、跨题冲突、执行遗漏、标签—证据不匹配和模板化风险。题型分布只观察，不重排案例、不加补偿权重、不自动调参。

## 兼容与安全

旧 `FROZEN-PREDICTION-V1` 保持只读并沿用原哈希算法。新冻结使用包含全案盲态模型、跨题检查、复训声明及全部逐题结构的整体哈希。

公开包不包含答案、答案映射或密钥。评分仍只能发生在冻结之后；无效 Issue 在启动轮次前完成预检，不会消费活动案例。简洁的用户展示使用 `public_summary`，不允许因此省略内部结构化推理。
