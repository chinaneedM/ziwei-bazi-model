# Open-source migration receipts

This directory records immutable engineering status objects for the transition
to a complete open-source, single-public-repository architecture.

A receipt in this directory may prove that code, policy, documentation, or
engineering tests were produced. It does not by itself grant formal training or
release permission.

Formal release additionally requires:

- repository visibility `public`;
- configured `FORTUNE_PUBLIC_ANSWER_KEY`;
- passing public GitHub Actions;
- exact public-distribution manifests for every active knowledge/data pack;
- a synthetic public-envelope end-to-end PASS receipt;
- merged and activated release bindings.
